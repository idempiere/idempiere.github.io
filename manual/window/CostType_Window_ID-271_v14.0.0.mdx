## Tab: Cost Type

*Tab Level 0 · Created 01/06/2003 · Updated 24/04/2005*

**Description:** Type of Cost (e.g. Current, Plan, Future)

**Comment/Help:** You can define multiple cost types. The cost type selected in an Accounting Schema is used for accounting.

| **Name** | **Description** | **Comment/Help** | **Technical Data** |
|---|---|---|---|
| Tenant | Tenant for this installation. | A Tenant is a company or a legal entity. You cannot share data between Tenants. | M_CostType.AD_Client_ID<small> numeric(10) <br/> Table Direct</small> |
| Organization | Organizational entity within tenant | An organization is a unit of your tenant or legal entity - examples are store, department. You can share data between organizations. | M_CostType.AD_Org_ID<small> numeric(10) <br/> Table Direct</small> |
| Name | Alphanumeric identifier of the entity | The name of an entity (record) is used as an default search option in addition to the search key. The name is up to 60 characters in length. | M_CostType.Name<small> character varying(60) <br/> String</small> |
| Description | Optional short description of the record | A description is limited to 255 characters. | M_CostType.Description<small> character varying(255) <br/> String</small> |
| Comment/Help | Comment or Hint | The Help field contains a hint, comment or help about the use of this item. | M_CostType.Help<small> character varying(2000) <br/> Text</small> |
| Active | The record is active in the system | There are two methods of making records unavailable in the system: One is to delete the record, the other is to de-activate the record. A de-activated record is not available for selection, but available for reports.<br/>There are two reasons for de-activating and not deleting records:<br/>(1) The system requires the record for audit purposes.<br/>(2) The record is referenced by other records. E.g., you cannot delete a Business Partner, if there are invoices for this partner record existing. You de-activate the Business Partner and prevent that this record is used for future entries. | M_CostType.IsActive<small> character(1) <br/> Yes-No</small> |
