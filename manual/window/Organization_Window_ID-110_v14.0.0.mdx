# Organization

<span className="badge badge--primary">Window</span> <span className="badge badge--secondary">ID 110</span>

*11/06/1999 → 02/01/2000*

![Organization](/img/manual/Organization-Window_iDempiere_v14.0.0.png)

**Description:** Maintain Organizations

**Comment/Help:** The Organization Window allows you to define and maintain Organizational entities.  An Organization is often a legal entity or sub-unit for which documents and transactions are processed

## Tab: Organization

*Tab Level 0 · Created 11/06/1999 · Updated 24/02/2018*

**Description:** Define Organizations

**Comment/Help:** The Organization Tab is used to define an Organization.  Each Organization has a Key and Name and optionally a Description.  
When adding a new organization, you must re-login to be able to access the new organization.

| **Name** | **Description** | **Comment/Help** | **Technical Data** |
|---|---|---|---|
| Tenant | Tenant for this installation. | A Tenant is a company or a legal entity. You cannot share data between Tenants. | AD_Org.AD_Client_ID<small> numeric(10) <br/> Table Direct</small> |
| Search Key | Search key for the record in the format required - must be unique | A search key allows you a fast method of finding a particular record.<br/>If you leave the search key empty, the system automatically creates a numeric number.  The document sequence used for this fallback number is defined in the "Maintain Sequence" window with the name "DocumentNo_&lt;TableName&gt;", where TableName is the actual name of the table (e.g. C_Order). | AD_Org.Value<small> character varying(40) <br/> String</small> |
| Name | Alphanumeric identifier of the entity | The name of an entity (record) is used as an default search option in addition to the search key. The name is up to 60 characters in length. | AD_Org.Name<small> character varying(60) <br/> String</small> |
| Description | Optional short description of the record | A description is limited to 255 characters. | AD_Org.Description<small> character varying(255) <br/> String</small> |
| Active | The record is active in the system | There are two methods of making records unavailable in the system: One is to delete the record, the other is to de-activate the record. A de-activated record is not available for selection, but available for reports.<br/>There are two reasons for de-activating and not deleting records:<br/>(1) The system requires the record for audit purposes.<br/>(2) The record is referenced by other records. E.g., you cannot delete a Business Partner, if there are invoices for this partner record existing. You de-activate the Business Partner and prevent that this record is used for future entries. | AD_Org.IsActive<small> character(1) <br/> Yes-No</small> |
| Summary Level | This is a summary entity | A summary entity represents a branch in a tree rather than an end-node. Summary entities are used for reporting and do not have own values. | AD_Org.IsSummary<small> character(1) <br/> Yes-No</small> |
| Replication Strategy | Data Replication Strategy | The Data Replication Strategy determines what and how tables are replicated  | AD_Org.AD_ReplicationStrategy_ID<small> numeric <br/> Table Direct</small> |
