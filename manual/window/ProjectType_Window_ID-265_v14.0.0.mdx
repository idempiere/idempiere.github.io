## Tab: Project Type

*Tab Level 0 · Created 28/05/2003 · Updated 02/01/2000*

**Description:** Maintain Type of a Project

**Comment/Help:** Type of the project with optional phases and tasks of the project with standard performance information

| **Name** | **Description** | **Comment/Help** | **Technical Data** |
|---|---|---|---|
| Tenant | Tenant for this installation. | A Tenant is a company or a legal entity. You cannot share data between Tenants. | C_ProjectType.AD_Client_ID<small> numeric(10) <br/> Table Direct</small> |
| Organization | Organizational entity within tenant | An organization is a unit of your tenant or legal entity - examples are store, department. You can share data between organizations. | C_ProjectType.AD_Org_ID<small> numeric(10) <br/> Table Direct</small> |
| Name | Alphanumeric identifier of the entity | The name of an entity (record) is used as an default search option in addition to the search key. The name is up to 60 characters in length. | C_ProjectType.Name<small> character varying(60) <br/> String</small> |
| Description | Optional short description of the record | A description is limited to 255 characters. | C_ProjectType.Description<small> character varying(255) <br/> String</small> |
| Comment/Help | Comment or Hint | The Help field contains a hint, comment or help about the use of this item. | C_ProjectType.Help<small> character varying(2000) <br/> Text</small> |
| Active | The record is active in the system | There are two methods of making records unavailable in the system: One is to delete the record, the other is to de-activate the record. A de-activated record is not available for selection, but available for reports.<br/>There are two reasons for de-activating and not deleting records:<br/>(1) The system requires the record for audit purposes.<br/>(2) The record is referenced by other records. E.g., you cannot delete a Business Partner, if there are invoices for this partner record existing. You de-activate the Business Partner and prevent that this record is used for future entries. | C_ProjectType.IsActive<small> character(1) <br/> Yes-No</small> |
| Project Category | Project Category | The Project Category determines the behavior of the project:<br/>General - no special accounting, e.g. for Presales or general tracking<br/>Service - no special accounting, e.g. for Service/Charge projects<br/>Work Order - creates Project/Job WIP transactions - ability to issue material<br/>Asset - create Project Asset transactions - ability to issue material | C_ProjectType.ProjectCategory<small> character(1) <br/> List</small> |
