**Goal:** Functional

**Developers:** Deepak Pansheriya, Carlos Ruiz

**Description:**

Now you can define a tree for any table that has *Summary Level* field (column name *IsSummary*).

Simply define the tree as usual on the Tree window, like this:

![01 TreeOnAnyTable](pathname:///img/new-features/v3.0/01_TreeOnAnyTable.png)

**TIP:** You can also enable the checkbox *Has Tree* on the corresponding tab.

And enjoy the new tree feature:

![02 TreeOnAnyTable](pathname:///img/new-features/v3.0/02_TreeOnAnyTable.png)

**TIP:** Your new tree can also be [Driven by Search Key](https://wiki.idempiere.org/en/NF2.1_Tree_Driven_by_Search_Key)

**Parent Column:** Version 8.2 introduce Parent Column ([tiket](https://idempiere.atlassian.net/browse/IDEMPIERE-5008)), which allow define for tenant admin the column from custom table. This improve performance.

**Technical Info:** [IDEMPIERE-2675](http://idempiere.atlassian.net/browse/IDEMPIERE-2675)
